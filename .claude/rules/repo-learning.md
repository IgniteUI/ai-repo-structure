# Learning repo rule

This repository is a teaching repo about how to structure AI-enabled repositories.

When working on matching files:

- Treat the real repository files as the source of truth.
- If you change an example tree or example path, update the real file structure or correct the example immediately.
- Prefer concrete, working examples from this repository over generic placeholders.
- Keep examples short, copy-pasteable, and easy to scan.
- Do not invent unsupported folders, frontmatter fields, or conventions.
- Keep Claude-side and GitHub-side examples aligned when they represent equivalent concepts.
- Keep naming consistent across agent names, skill names, and folder names.
- When in doubt, follow the existing patterns in the repository or ask for clarification.
